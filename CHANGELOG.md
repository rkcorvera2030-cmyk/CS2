# CHANGELOG

All notable changes to this project will be documented in this file.  
The format follows *Semantic Versioning*: vMAJOR.MINOR.PATCH  
- *MAJOR*: Significant changes or new modules  
- *MINOR*: New features or improvements  
- *PATCH*: Small fixes or adjustments  

---

## [v1.0.0]
- Initial project proposal completed.
- Added project objectives, planned features, and logic plan.

## [v1.1.0]
- Created basic program structure and main menu.
- Added "Add Expense" feature to record amount, date, category, and notes.

## [v1.2.0]
- Implemented "View Expenses" feature showing all recorded entries.
- Added automatic total expense calculation.

## [v1.3.0]
- Added "Search Expenses" feature by category or date.
- Improved menu navigation.

## [v1.3.1]
- Fixed bug where expenses with no category caused an error.
- Adjusted output formatting for better readability.

## [v1.4.0]
- Added monthly spending report feature.
- Optimized expense storage for faster loading.

## [v1.5.0]
- Added "Edit Expense" feature allowing users to modify existing entries.
- Implemented "Delete Expense" option for removing incorrect or unwanted records.
- Improved data validation to prevent incomplete expense entries.

## [v1.5.1]
- Fixed issue where edited expenses did not update in the monthly report.
- Improved error messages for invalid date formats.

## [v1.6.0]
- Added data export feature (CSV format).
- Implemented basic data backup system to prevent accidental loss.

## [v1.7.0]
- Added budget-setting feature for weekly and monthly limits.
- Created automatic alerts when spending exceeds the set budget.

## [v1.7.1]
- Fixed incorrect budget calculations when editing past expenses.
- Improved UI spacing and clarity in the reports section.

## [v1.8.0]
- Added category-based charts for visual spending analysis.
- Enhanced search function to support combined queries (date + category).

## [v2.0.0]
- Complete UI redesign for a cleaner and more modern interface.
- Refactored codebase for better scalability and performance.
- Introduced modular architecture for easier future feature additions.
